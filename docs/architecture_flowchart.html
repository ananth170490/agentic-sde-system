Workflow explanation

The system turns one plain-English requirement into a validated, human-approved engineering outcome by moving a single shared RunState object through a fixed sequence of agents, with three points where a human must sign off before the run continues.

1. Intake and ambiguity check

The requirement is parsed into explicit requirements, implicit requirements, and open ambiguities, each carrying an ambiguity score. If that score is high, the run pauses at the clarify gate and waits for a human to answer specific questions before continuing — the system never guesses its way past a genuinely unclear requirement.

2. Context-aware design

Once the requirement is unambiguous, brownfield requests first go through the codebase reasoning agent, which indexes the existing repository and identifies impacted files and modules. Every request — brownfield or greenfield — then reaches the architect agent, which produces the components, data model, API contract, and trade-offs.

3. Planning and the first approval gate

The task decomposer agent turns the design into a dependency-aware task DAG, with each task's file ownership defined up front to prevent collisions. This plan is shown to a human at the plan approval gate — execution does not begin until it's approved.

4. Execution loop, with built-in recovery

Approved tasks run in dependency order. Each task passes through CodeGen → TestGen → Validation, where validation actually executes pytest, py_compile, and static analysis rather than trusting a model's self-report. A failed task is repaired using the validation feedback and retried, up to a fixed retry limit — only after those retries are exhausted does the run pause for human intervention on that specific task. This loop is what gives the system real error handling and recovery, not just linear execution.

5. Risk synthesis and the second and third gates

Once every task is done, the risk and docs agent compiles risks, trade-offs, and a structured engineering summary. The run then pauses at the merge approval gate for a final human sign-off before being marked complete, with the full RunState persisted at every step so a crashed run can resume from wherever it left off rather than starting over.
